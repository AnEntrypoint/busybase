Why BusyBase

Everything you need.
Nothing you don't.

Built for developers who want Supabase ergonomics without the operational overhead of Postgres, Docker, and microservices.

⚡
Bun-native Performance
Built entirely on Bun — sub-millisecond startup, native TypeScript, no transpile step. Ships as a single self-contained binary for Linux, macOS, and Windows.
bun serve
🔌
Drop-in Supabase SDK
Identical { data, error } response shape, same method names — .from().select().eq(). Migrate existing Supabase projects with a one-line URL change.
Supabase JS v2 compatible
🔑
Ed25519 Keypair Auth
Anonymous-first auth using WebCrypto Ed25519 — no passwords needed. Users get a cryptographic identity instantly. Optionally upgrade to email/password later. Zero external dependencies.
WebCrypto · zero deps
🧠
Native Vector Search
Store embeddings alongside your data. Query by semantic similarity with .vec(embedding, limit). Works with OpenAI, Ollama, or any embedding model — bring your own vectors.
LanceDB ANN · Apache Arrow
🪝
Pluggable Hooks System
18 lifecycle hooks — from onRequest middleware to beforeInsert / canAccess row-level security. Point BUSYBASE_HOOKS to a TypeScript file and you're done.
BUSYBASE_HOOKS=./hooks.ts
📁
File-Based Storage
LanceDB stores data as Apache Arrow files — no database server process. Data lives in a directory you own. Back it up with cp -r. No schema migrations needed — tables are created on first insert.
LanceDB · Arrow format
📧
Zero-Code Email (SMTP)
Password reset emails sent via Bun TCP SMTP — configure with five environment variables. Override entirely with a custom sendEmail hook to use any provider (Resend, SendGrid, etc.).
BUSYBASE_SMTP_*
🌐
Full REST API
Supabase-compatible REST endpoints with filter operators (eq, neq, gt, lt, like, in, or, not), Content-Range pagination, Prefer: return=minimal, and CORS included.
REST · CORS · PostgREST-style
🛠
First-Class CLI
Every operation is a CLI command — busybase insert, query, vec, signup. The CLI is the test runner — busybase test runs a full SDK e2e suite against your live server.
busybase serve · test · query
Quick Start

From zero to running
in under 60 seconds.

No config files. No database setup. No Docker. Just run and build.

1
Start the server
Run directly with Bun, or download a standalone binary from Releases — no runtime required.
bunx busybase serve
2
Install the SDK
Install the client package or import the SDK file directly. Fully compatible with Supabase JS v2.
bun add busybase
3
Connect and query
Same API as Supabase — .from().select().eq() just works.
4
Run the test suite
Verify everything works end-to-end with the built-in SDK test runner.
bunx busybase test 
import BB from "busybase";

const db = BB("http://localhost:54321", "local");

// Insert (auto-creates table)
const { data, error } = await db
  .from("todos")
  .insert({ title: "Buy milk", done: false });

// Query with filters
const { data: todos } = await db
  .from("todos")
  .select("*")
  .eq("done", false)
  .order("title", { ascending: true })
  .limit(20);

// Update
await db.from("todos")
  .update({ done: true })
  .eq("title", "Buy milk");

// Delete
await db.from("todos")
  .delete()
  .eq("done", true);
// Anonymous-first — no email needed!
const { data } = await db.auth.keypair.signIn();
// data.user.id — persistent cryptographic identity
// data.session.access_token

// Backup your keypair
const keys = db.auth.keypair.export();
// { privkey, pubkey } — save these!

// Restore on new device
await db.auth.keypair.restore(keys.privkey, keys.pubkey);
// Same user.id ✓

// Or use email/password
await db.auth.signUp({ email, password });
const { data: { session } } =
  await db.auth.signInWithPassword({ email, password });

// Upgrade keypair account with email later
await db.auth.updateUser({
  email: "user@example.com",
  data: { name: "Alice" }
});
// Store embeddings with your data
await db.from("articles").insert([
  { title: "Cats",  vector: [0.9, 0.1, 0.0, 0.0] },
  { title: "Dogs",  vector: [0.1, 0.9, 0.0, 0.0] },
  { title: "Fish",  vector: [0.0, 0.0, 1.0, 0.0] },
]);

// Semantic similarity search
const { data } = await db
  .from("articles")
  .select("*")
  .vec([0.85, 0.15, 0.0, 0.0], 2);
// data[0].title === "Cats"
// data[0]._distance — similarity score

// Works with real embeddings too
import OpenAI from "openai";
const embedding = await openai.embeddings.create({
  model: "text-embedding-3-small", input: query
});
const { data: results } = await db
  .from("articles")
  .select("title,url")
  .vec(embedding.data[0].embedding, 5);
# Start server (port 54321)
busybase serve

# Run full SDK test suite
busybase test

# User management
busybase signup user@example.com mypassword
busybase signin user@example.com mypassword

# CRUD
busybase insert todos '{"title":"Buy milk","done":false}'
busybase query  todos done=false
busybase update todos '{"done":true}' title=Buy\ milk
busybase delete todos title=Buy\ milk

# Vector search
busybase vec articles '[0.9,0.1,0.0,0.0]' 5

# With env vars
BUSYBASE_URL=https://api.myapp.com \
BUSYBASE_PORT=8080 \
BUSYBASE_DIR=./data \
busybase serve
// hooks.ts — set BUSYBASE_HOOKS=./hooks.ts
export const canAccess = ({ user, table, method }) => {
  if (!user && method !== "GET")
    return { error: "Login required" };
};

export const beforeInsert = (table, rows) => {
  if (table === "comments")
    rows.forEach(r => r.created_at = new Date().toISOString());
};

export const onSignup = async (user) => {
  await sendWelcomeEmail(user.email);
};

export const sendEmail = async ({ to, subject, html }) => {
  // Use Resend, SendGrid, etc.
  await resend.emails.send({ from, to, subject, html });
};
